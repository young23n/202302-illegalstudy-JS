1. Ajax에 대해서 알아오시오
    AJAX란 자바스크립트의 라이브러리 중 하나입니다.
    Asynchronous Javascript And Xml(비동기식 자바스크립트와 xml)의 약자입니다.
    웹 페이지 전체를 다시 로딩하지 않고 웹 페이지의 일부분만을 갱신하는 기법입니다.
    덕분에 기존에 불가능하던 UI와 자원낭비를 방지할 수 있습니다.
    
2. jQuery에 대해서 알아오시오
    HTML의 DOM 조작과 이벤트 제어, 애니메이션 그리고 Ajax까지 웹 화면을 다루는 
    자바스크립트 라이브러리입니다. 작은 양의 코드로도 많은 일을 할 수 있게 설계되어 있습니다.

3. V8 엔진에 대해서 알아오시오
    V8 엔진은 Google이 개발한 오픈소스로 가장 대중적인 자바스크립트 엔진입니다.

4. Nodejs에 대해서 알아오시오
    Node.js는 Chrome V8 JavaScript 엔진으로 빌드 된 JavaScript 런타임(환경)입니다.
    JavaScript 를 웹 브라우저에서 독립시킨 것으로 Node.js를 설치하게 되면 터미널프로그램(윈도우의 cmd, 맥의 terminal 등)에서 
    Node.js를 입력하여 브라우저 없이 바로 실행할 수 있다. 이렇게 Node.js를 이용하여 웹 브라우저와 무관한 프로그램을 만들 수 있게 되었다.
    중요한 것은 Node.js를 이용하여 서버를 만들 수 있다는 것이다.
    이전까지 Server-Client 웹사이트를 만들 때 웹에서 표시되는 부분은 JavaScript 를 사용하여 만들어야만 했으며, 서버는 Reby, Java 등 다른 언어를 써서 
    만들었어야 했는데 마침내 한 가지 언어로 전체 웹 페이지를 만들 수 있게 된 것이다.

5. SPA 프레임워크의 뜻을 알아오시오
    SPA란 Single Page Application(단일 페이지 어플리케이션)의 약자입니다.
    기존 웹 서비스는 요청시마다 서버로부터 리소스들과 데이터를 해석하고 화면에 렌더링하는 방식입니다.
    SPA형태는 브라우저에 최초에 한번 페이지 전체를 로드하고, 이후부터는 특정 부분만 Ajax를 통해 데이터를 
    바인딩하는 방식입니다.

V8 엔진에 대해서 정리해보시오
    C++로 개발하였고 Chrome 브라우저에서 사용되고 있습니다. v8엔진은 인터프리터를 사용하는 대신 
    JIT(Just In Time) 컴파일러를 구현함으로써 코드 실행 시에 자바스크립트 코드를 기계어로 컴파일합니다
    V8 엔진은 오픈 소스 프로젝트로, 소스 코드가 공개되어 있어 누구나 자유롭게 사용하고 수정할 수 있습니다.

아래의 코드들을 실행한 값을 알아오시오. 그리고 그 이유도
    1.  undefined
        11
    
    2.  undefined
        10
    
    두 코드 모두 호이스팅에 대해 알려주려는 것 같습니다.
    변수가 선언된 위치에 상관없이 인터프리터에 의해 코드 전체가 읽히고
    변수는 모두 최상단으로 올라갑니다.
    이때 초기화되지 않은 변수는 undefined가 할당됩니다.
